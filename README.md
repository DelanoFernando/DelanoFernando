I work in Marketing Operations, which means I spend my time making sure the systems behind go-to-market teams actually work the way they're supposed to.

My background is in software engineering. I've written code across enough stacks to know when something can be automated, when a data model is going to cause problems six months from now, and when the right fix is a properly written SQL query rather than another meeting. That technical foundation is what makes the operational work useful: not just managed, but genuinely improved.

I care about the work being right. I enjoy collaborating with both technical and non-technical teams, and I've found the most interesting problems usually sit at the boundary between them.
